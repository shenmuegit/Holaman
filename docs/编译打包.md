# Holaman 编译打包文档

本文档说明 Holaman 在 Windows 11 与 macOS Apple Silicon 上的开发、编译和安装包构建流程。

## 1. 技术栈

- 桌面壳：Tauri 2
- 前端：React + TypeScript + Vite
- 样式：Tailwind CSS
- 后端：Rust + reqwest + sqlx + SQLite
- 包管理：pnpm

## 2. 环境要求

### 2.1 通用要求

- Node.js 20 或更高版本。
- pnpm 10 或更高版本。
- Rust stable。
- Tauri CLI，项目已通过 `@tauri-apps/cli` 安装到本地依赖中。

安装依赖：

```powershell
pnpm install
```

检查前端环境：

```powershell
node --version
pnpm --version
```

检查 Tauri / Rust 环境：

```powershell
pnpm tauri info
```

如果提示 `rustc: not installed` 或 `Cargo: not installed`，需要先安装 Rust。

Rust 安装地址：

```text
https://rustup.rs/
```

### 2.1.1 Cargo 代理

本项目已在 `.cargo/config.toml` 中配置 Cargo 下载代理：

```toml
[http]
proxy = "http://127.0.0.1:1080"
```

如果你的代理端口变化，修改该文件即可。也可以临时在 PowerShell 中设置：

```powershell
$env:HTTP_PROXY="http://127.0.0.1:1080"
$env:HTTPS_PROXY="http://127.0.0.1:1080"
```

### 2.2 Windows 11 要求

- Windows 11。
- Microsoft Edge WebView2 Runtime。
- Visual Studio 2022 Build Tools 或 Visual Studio 2022。
- MSVC C++ 构建工具。
- WiX Toolset 相关能力由 Tauri 打包流程使用，用于生成 `.msi`。

当前项目打包目标包含：

- `.msi`

### 2.3 macOS Apple Silicon 要求

- Apple Silicon Mac。
- macOS 12 或更高版本。
- Xcode Command Line Tools。
- Rust stable，目标架构为 `aarch64-apple-darwin`。

安装 Xcode Command Line Tools：

```bash
xcode-select --install
```

确认 Rust 目标：

```bash
rustup target list --installed
```

如果没有 `aarch64-apple-darwin`：

```bash
rustup target add aarch64-apple-darwin
```

当前项目打包目标包含：

- `.dmg`
- `.app`

## 3. 开发运行

### 3.1 仅运行前端预览

用于调试 React 界面。此模式不启动 Rust 后端，前端会使用浏览器预览模拟响应。

```powershell
pnpm dev --host 127.0.0.1
```

访问：

```text
http://127.0.0.1:1420/
```

### 3.2 运行 Tauri 桌面开发版

需要 Rust / Cargo 可用。

```powershell
pnpm tauri:dev
```

该命令会：

- 启动 Vite dev server。
- 编译 Rust 后端。
- 打开 Holaman 桌面窗口。
- 通过 Tauri Commands 调用真实 Rust Core。

## 4. 前端构建

只构建 Web 前端产物：

```powershell
pnpm build
```

构建产物目录：

```text
dist/
```

当前已验证：

```text
pnpm build
```

可以成功完成 TypeScript 检查与 Vite 构建。

## 5. 桌面安装包构建

### 5.1 Windows 11 打包

在 Windows 11 上执行：

```powershell
pnpm tauri:build
```

构建完成后，产物通常位于：

```text
src-tauri/target/release/bundle/msi/
```

预期产物：

```text
Holaman_0.1.0_x64_en-US.msi
```

实际文件名可能受 Tauri、WiX、系统区域语言和版本号影响。

### 5.2 macOS Apple Silicon 打包

在 Apple Silicon Mac 上执行：

```bash
pnpm install
pnpm tauri:build
```

构建完成后，产物通常位于：

```text
src-tauri/target/release/bundle/dmg/
src-tauri/target/release/bundle/macos/
```

预期产物：

```text
Holaman_0.1.0_aarch64.dmg
Holaman.app
```

实际文件名可能受 Tauri 版本和签名配置影响。

## 6. 当前 Tauri 配置

配置文件：

```text
src-tauri/tauri.conf.json
```

关键配置：

- 应用名称：`Holaman`
- 应用标识：`com.meolord.holaman`
- 默认窗口：`1280 x 820`
- 最小窗口：`1024 x 700`
- 前端开发地址：`http://localhost:1420`
- 前端构建目录：`../dist`
- 打包目标：`msi`、`dmg`、`app`
- macOS 最低版本：`12.0`
- Windows WiX 语言：`zh-CN`

## 7. 已知限制

- 当前 Windows 开发机如果未安装 Rust / Cargo，只能运行 `pnpm dev` 和 `pnpm build`，不能运行 `pnpm tauri:dev` 或 `pnpm tauri:build`。
- macOS `.dmg` 和 `.app` 需要在 macOS 上构建。
- 第一阶段尚未配置代码签名、公证和自动更新。
- 当前 bundle 图标仍使用 Tauri 默认行为，后续需要补正式图标资源。

## 8. 推荐发布前检查

发布安装包前执行：

```powershell
pnpm build
pnpm tauri info
pnpm tauri:build
```

检查项：

- 应用能启动并进入中文请求工作台。
- `GET` 请求能发送并显示响应。
- `POST JSON` 请求能发送并显示响应。
- History 可以写入并重新加载。
- Collection 可以保存并重新加载。
- 默认窗口尺寸符合设计稿。
- 页面没有装饰性渐变、卡片阴影和第二品牌色。
