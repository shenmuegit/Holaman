# Holaman 界面设计稿

## 1. 产品定位与设计转译

Holaman 是一个本地优先的桌面端 API 调试工具，第一屏必须直接进入可工作的 API 调试台，而不是营销页。视觉语言参考 `.codex/DESIGN.md` 的 Apple 风格：界面克制、留白明确、交互色单一、层级通过表面颜色和排版建立。

API 调试工具需要高密度信息扫描，因此 Holaman 不直接复制大面积产品摄影和低密度官网结构，而是将设计规范转译为安静、清晰、少装饰的工作台：

- 主体是工作区，不是宣传内容。
- 交互色只使用 Action Blue `#0066cc`。
- 页面通过白色、浅灰、近黑区块形成节奏。
- 卡片仅用于工具面板、列表项和独立功能块。
- 不使用装饰性渐变、装饰性阴影或发光效果。
- 阴影只预留给未来真实产品图或导入引导图，不用于按钮、卡片和文字。
- 所有关键操作保持 pill 形态，所有工具面板保持轻边框和 18px 圆角。

### 2.3 间距、圆角与深度

- 基础间距：8px。
- 面板内边距：16px 或 24px。
- 主区域 gutter：20px 到 24px。
- 全局导航高度：44px。
- 子导航高度：52px。
- 桌面工作台最小推荐高度：720px。

圆角：

- 全屏区域：0px。
- 紧凑按钮、树节点 hover：8px。
- 输入框和次级胶囊：11px。
- 工具面板和列表项：18px。
- 主操作、搜索、筛选 chip：9999px。

深度：

- 卡片和按钮不使用阴影。
- 面板边界使用 1px `#e0e0e0`。
- 粘性导航使用 `backdrop-filter: saturate(180%) blur(20px)`。
- 仅在未来出现真实产品渲染图时允许使用 `rgba(0, 0, 0, 0.22) 3px 5px 30px 0`。

## 3. 应用框架布局

Holaman 使用两层顶部导航 + 三栏工作台 + 底部响应区。

### 3.1 顶部全局导航

高度 44px，背景 `#000000`，文字白色。左侧显示 Holaman，中央显示顶级模块，右侧显示全局搜索、设置、当前工作区状态。

桌面导航项：

- Workspace
- Collections
- Environments
- Mock
- Runner
- Settings

移动或窄屏时，导航折叠为：

- 左侧 Holaman。
- 中央当前模块名。
- 右侧搜索按钮和菜单按钮。

### 3.2 毛玻璃子导航

高度 52px，背景 `rgba(245, 245, 247, 0.8)`，使用 backdrop blur。左侧显示当前页面名，例如 `Request Workspace`。右侧提供轻量操作：

- Import
- Export
- New Request
- Save

`Send` 不放在子导航，必须跟随 URL 输入栏，减少视线移动。

### 3.3 主工作台布局

桌面端默认布局：

```text
-------------------------------------------------------------+
| Global Nav 44px                                             |
+-------------------------------------------------------------+
| Frosted Sub Nav 52px                                        |
+--------------+-------------------------------+--------------+
| Left Rail    | Request Editor                | Right Panel  |
| 280px        | flexible                      | 320px        |
|              |                               |              |
+--------------+-------------------------------+--------------+
| Response Viewer                                             |
| 280px - 40vh                                                |
+-------------------------------------------------------------+
```

默认高度分配：

- 顶部导航固定。
- 中央请求编辑区占剩余高度的 58%。
- 底部响应区占剩余高度的 42%。
- 响应区可上下拖拽，最小 220px，最大 70vh。

## 4. 核心页面结构

### 4.1 Request Workspace

默认首页。用户打开应用后直接看到一个空白请求，而不是欢迎页。

区域组成：

- 左侧栏：Collections、History、Environments 三个切换标签。
- 中央上方：请求名称、方法、URL、Send。
- 中央主体：请求配置 Tabs。
- 右侧面板：环境变量预览、鉴权摘要、文档辅助。
- 底部：响应查看。

空状态：

- 没有请求时仍显示完整请求编辑器。
- URL 输入框 placeholder 为 `https://api.example.com/users`。
- 左侧 Collection 为空时显示 `Create Collection` pill 按钮。

### 4.2 Collections

用于请求集合管理，不作为单独大页面打断调试流程。左侧栏中的 Collection 树是主要入口，顶级模块页用于批量管理。

页面内容：

- 顶部搜索输入，pill 形态。
- Collection 网格，使用 18px 圆角白色工具卡。
- 每个 Collection 显示名称、请求数、最近更新时间。
- 进入 Collection 后显示文件夹树、请求列表、变更历史。

### 4.3 Environments

用于管理开发、测试、预发布、生产环境。

页面内容：

- 左侧环境列表。
- 中央变量表格。
- 右侧敏感变量说明和当前生效变量预览。

变量表格列：

- Key
- Initial Value
- Current Value
- Secret
- Enabled
- Description

敏感值默认显示为圆点或 `••••••`，复制和导出需要明确用户操作。

### 4.4 Mock Server

用于本地 Mock 服务管理。

页面内容：

- 顶部状态条：端口、运行状态、请求数量。
- 左侧 Mock 规则列表。
- 中央规则编辑器。
- 右侧请求日志。

主操作：

- Start
- Stop
- New Rule
- Copy Local URL

运行状态不引入新的绿色。使用 `Running` 文案 + Action Blue 标签表示运行，`Stopped` 使用灰色标签。

### 4.5 Runner

用于 Collection 批量测试。

页面内容：

- 左侧选择 Collection 和 Environment。
- 中央运行配置。
- 底部或右侧显示运行结果。

运行配置：

- Iterations
- Delay
- Data File
- Stop on Failure
- Save Response

测试结果：

- 请求名。
- 状态码。
- 耗时。
- 断言通过数。
- 失败信息。

### 4.6 GraphQL

GraphQL 是请求类型的一种，不单独脱离 Workspace。

入口：

- Body Tab 中选择 `GraphQL`。
- 或 New Request 菜单选择 `GraphQL Request`。

布局：

- Query 编辑器。
- Variables 编辑器。
- Schema / Docs 右侧辅助面板。
- Response 使用统一响应查看器。

### 4.7 WebSocket / SSE

WebSocket 和 SSE 使用专门的实时调试布局。

布局：

- 顶部连接栏：协议、URL、Connect / Disconnect。
- 左侧消息历史。
- 中央消息查看。
- 底部发送输入区。
- 右侧 Headers、Params、连接状态。

消息气泡不使用彩色聊天样式，使用白色或浅灰工具行，靠标签区分 `sent` 和 `received`。

### 4.8 Settings

设置页采用浅灰背景和白色工具卡。

分区：

- Network：代理、超时、SSL 证书校验。
- Storage：SQLite 本地数据路径、导入导出。
- Appearance：浅色主题优先，暗色代码区可配置。
- Security：敏感变量加密、导出遮罩。
- About：版本、平台、更新状态。

## 5. 请求编辑区设计

### 5.1 请求标题行

高度 44px 到 52px。

内容：

- 请求名称，默认 `Untitled Request`。
- 保存状态：`Saved` / `Unsaved`。
- 所属 Collection 路径。
- 复制、重命名、更多操作按钮。

请求名称使用 `bodyStrong`。辅助信息使用 `finePrint`。

### 5.2 方法、URL 与 Send

布局：

```text
[ METHOD v ] [ https://api.example.com/users                  ] [ Send ]
```

规格：

- 方法选择器宽 116px，高 44px，圆角 11px，白底，1px hairline。
- URL 输入框高度 44px，pill 圆角，白底，1px hairline。
- Send 按钮高度 44px，pill 圆角，背景 `#0066cc`，文字白色。
- 发送中按钮文案为 `Sending`，按钮保持蓝色，左侧显示轻量进度指示。

方法不使用多色标签，统一黑色文字。危险方法通过确认交互和文案提示处理，不通过红色强调。

### 5.3 请求配置 Tabs

Tabs：

- Params
- Headers
- Body
- Auth
- Cookies
- Scripts
- Tests
- Docs

Tabs 使用 14px 字体，选中态为 Action Blue 下划线或浅蓝文字，不使用厚重背景块。

### 5.4 Params

表格列：

- Enabled
- Key
- Value
- Description

行为：

- 勾选启用或禁用参数。
- 自动同步 URL query。
- 空行始终保留在底部。
- 支持 Bulk Edit。

### 5.5 Headers

表格列：

- Enabled
- Key
- Value
- Description

行为：

- 常用 Header 自动补全。
- 敏感 Header 可手动标记隐藏。
- 重复 Header 给出非阻断提示。

### 5.6 Body

类型切换：

- none
- JSON
- form-data
- x-www-form-urlencoded
- raw
- XML
- binary
- GraphQL

JSON、raw、XML、GraphQL 使用 Monaco Editor。form-data 和 x-www-form-urlencoded 使用表格。binary 使用文件选择器。

编辑器区域：

- 背景白色。
- 字体 13px 代码字体。
- 行高 1.55。
- 工具栏提供 Format、Copy、Clear。

### 5.7 Auth

鉴权方式：

- No Auth
- Bearer Token
- Basic Auth
- API Key
- OAuth 2.0
- Digest Auth
- Inherit from Collection

Auth 面板使用白色工具卡，但不嵌套卡片。每种鉴权方式只展示当前需要字段。

### 5.8 Scripts 与 Tests

Scripts 包含：

- Pre-request Script。
- Post-response Script。

Tests 包含：

- 状态码断言。
- 响应字段断言。
- 响应时间断言。
- Header 断言。
- Body 包含断言。

脚本编辑器底部显示执行日志，日志区域使用 `#252527` 暗色背景，文字白色或浅灰，链接使用 `#2997ff`。

## 6. 响应查看区设计

### 6.1 响应摘要条

响应区顶部固定一行摘要：

```text
Status 200 OK    Time 128 ms    Size 14.2 KB    Saved from 10:24:33
```

规格：

- 高度 44px。
- 背景 `#f5f5f7`。
- 使用 14px 字体。
- 指标之间使用细分隔线。
- 状态码标签使用 9999px pill，颜色仍以蓝色或灰色为主，不引入状态色系统。

### 6.2 响应 Tabs

Tabs：

- Body
- Headers
- Cookies
- Tests
- Timeline
- Raw

Body 默认打开。Headers 和 Cookies 使用表格。Tests 显示断言结果。Timeline 显示 DNS、TCP、TLS、TTFB、Download 等阶段。

### 6.3 Body Viewer

模式：

- Pretty。
- Raw。
- Preview。
- Visualize。

JSON 响应默认 Pretty，支持折叠、搜索、复制路径。图片响应使用浅灰背景居中预览。HTML 响应默认源码查看，用户可切换 Preview。

错误响应不使用大面积红色背景，使用顶部摘要和错误说明区表达。

## 7. 左侧栏设计

左侧栏宽 280px，背景 `#f5f5f7`，右侧 1px hairline。

顶部为三段式切换：

- Collections
- History
- Environments

切换控件使用 pill 容器，选中态白底 + Action Blue 文本。

### 7.1 Collection 树节点

节点内容：

- 展开图标。
- 方法或文件夹图标。
- 请求名称。
- 可选状态标记。

规格：

- 高度 32px。
- 圆角 8px。
- 选中态白底，文字 `#1d1d1f`，左侧细蓝线或蓝色图标。
- hover 使用 `rgba(0, 0, 0, 0.04)`。

方法标签不使用彩虹色。统一使用细边框和方法文本。

### 7.2 History

历史列表按时间分组：

- Today。
- Yesterday。
- Earlier。

列表项显示方法、URL 摘要、状态码、耗时和时间。点击恢复请求，但不自动覆盖当前未保存请求。

### 7.3 Environment 快速预览

显示当前环境变量的 key 列表和已启用状态。敏感值不可直接显示。

## 8. 右侧辅助面板设计

右侧面板宽 320px，背景白色，左侧 1px hairline。

Tabs：

- Variables
- Auth
- Docs
- Console

### 8.1 Variables

显示当前请求可用变量，按来源分组：

- Global。
- Environment。
- Collection。
- Request。

每个变量显示 key、来源、当前值预览。敏感值遮罩。

### 8.2 Auth Summary

展示当前最终生效的鉴权来源和字段：

- Inherited from Collection。
- Bearer Token set。
- Header injection preview。

### 8.3 Docs

展示当前请求的文档草稿：

- 描述。
- Params。
- Headers。
- Body Schema。
- Response Examples。

Docs 面板可编辑，但编辑行为应轻量，不抢占主工作区。

### 8.4 Console

显示脚本日志、网络错误、Mock 命中记录。暗色背景使用 `#252527`，日志行无阴影、无彩色块。

## 10. 桌面平台适配

### 10.1 Windows 11

- 默认窗口尺寸：1280px × 820px。
- 最小窗口尺寸：1024px × 700px。
- 使用系统窗口标题栏，除非后续明确自绘标题栏。
- 字体优先使用 `Segoe UI` 作为 system-ui 回退。
- 文件路径、证书路径、SQLite 路径必须显示为 Windows 友好的可复制文本。
- 安装包目标为 `.msi`，可附加 `.exe`。

### 10.2 macOS Apple Silicon

- 默认窗口尺寸：1280px × 820px。
- 最小窗口尺寸：1024px × 700px。
- 字体走系统 SF Pro。
- 顶部全局导航下方留出平台窗口控制空间时，内容不得被红黄绿窗口按钮遮挡。
- 安装包目标为 Apple Silicon `.dmg` 和 `.app`。

### 10.3 跨平台一致性

- 主工作流、快捷键文案、导入导出结构保持一致。
- 平台差异只体现在系统字体、窗口装饰、路径展示和系统文件选择器。
- 所有布局必须在浅色主题下优先完成。

## 11. 响应式布局策略

### 1440px 及以上

- 内容最大宽度锁定在 1440px。
- 左侧栏 280px。
- 右侧栏 320px。
- 响应区可拖拽。
- 主工作区显示完整三栏。

### 1024px 到 1439px

## 12. 后续实现验收标准

界面实现完成后，应满足以下标准：

- 打开应用后直接进入 Request Workspace。
- 用户无需创建 Collection 即可发送一个临时请求。
- Send、Save、Run、Start Mock 等主操作全部使用 Action Blue pill。
- 不出现第二品牌色，不出现装饰性渐变。
- 卡片、按钮、文字不使用阴影。
- 代码区、日志区、深色面板只使用近黑色表面和 `#2997ff` 链接。
- Collection、Environment、Mock、Runner、GraphQL、WebSocket 都有明确入口。
- Windows 11 与 macOS Apple Silicon 的窗口尺寸、字体回退和安装包目标已在实现任务中体现。
- 1440px、1024px、834px、640px、419px 宽度下无明显遮挡和文字溢出。
- 所有敏感变量默认遮罩，导出时可排除敏感字段。
